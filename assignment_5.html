Introduction

Power BI is an interactive data visualization software that was developed by Microsoft with the focus on business intellegence.

Power BI is used for analyzing and visualizing raw data to present information. It combines business analytics, and data visualization to help businesses understand their data and make decisions from it. The core part of Power BI are the interactive dashboard that can be created from the data. The raw data is uploaded and then anaylzed or transformed as needed and loaded. From there, the users can create many different kinds of visualizations and dashboards. Power BI has multiple components such as Power BI Desktop, Service and Mobile.

It is a powerful tool that is very flexible to meet business needs. The interactivity between all of the visuals and data and how the data can be manipulated makes Power BI stand out from the other visualization tools. With features such as Power Querying, DAX (Data Analysis Expression), Visuals and Service, Power BI, makes for an excellent choice to presenting data in different ways all the way from very simple to complex.

Follow along with me and I will show you how to create a basic visual in Power BI!

Let's Get Started

System requirements

  • Windows 10, Windows Server 2012 R2, Windows Server 2012, Windows 8, Windows 8.1, Windows Server 2016, Windows Server 2019, Windows 11
  • Microsoft Power BI Desktop requires Internet Explorer 10 or greater.
  • Microsoft Power BI Desktop is available for 32-bit (x86) and 64-bit (x64) platforms.

To get started, be sure that your system meets the requirements outlined above. First, let's download Power BI. In this tutorial, we will be using Power BI Desktop. Go to Power BI Desktop Download, and select Download Free to download from the Microsoft Store or for other download options, select See download or language options which will give you setup.exe files.

While Power BI is downloading, make sure to create a Microsoft Account. If you already have one, please sign in. This will allow you to publish your report later.

Once you finish downloading and going through the install process, you should have Power BI correctly installed and ready to go!

First Time Launch

When you first open Power BI Desktop, it displays the Home Page. From here, you can start with a Blank Report or start with a Data Source. There will also be a section for Recommendations that you can select. It will display all recent Reports that you've been working on.

For this example, we will go ahead and select Blank Report. This will take us to a blank report and default report screen of Power BI. On the left side, there are icons for Power BI Desktop Views: Report, Data, Model, and DAX Query View. You can change the view at any time by clicking on the icons and is indicated by the vertical line to the left of the icon.

The Report view is the default view for Power BI reports.

For this example, we won't be worrying about Model View or DAX Querying View. You can find more detailed information about Model View and DAX Querying View on the Learn Microsoft Webpage.



Connecting to the Data

Now that we have Power BI Desktop instaled and we've looked at the Home Page and Blank Report views, we'll go ahead and connect some data!

To see all the possible types of data sources that we can upload to Power BI Desktop, select Get Data > More... in the Home tab, and in the Get Data window. Scroll through the list to explore all possible options!

For our example, let's use a 2024 Population Projections by Country dataset from Kaggle! Click HERE to download the .csv file.

Let's imagine we are a data analyst that is trying to look at world populations. You have a dataset that contains historical data as well as projected populations all across the world. You want to use this data to help you and your peers understand what these trends look like. Using the provided .csv file above, we can start populating our Power BI report with actual data.

On the Power BI Desktop Home tab, select Get Data > Text/CSV. From there, open the 2024Populations.csv file. Allow a moment for the Power BI to uplaod the file.


Another window will appear showing a preview of the data and a title 2024Populations.csv. This window is just previewing the data and giving you an idea as to what your data will look like in a Power BI table. At the top, you will see dropdown options for File Origin, Delimiter, and Data Type Detection. We will leave those as default options and let Power BI auto detect what type of data it will process and the data type that each column has.

From here, we can either Load the data as is, or Transform Data to make changes in the table before we load it in the report.

We will go ahead and click Transform Data to look over the data that Power BI has uploaded and confirm that we are happy with the data types and the stucture that our data is in.


When you select Transform Data, the Power Query Editor Launches, with a view of the table from our CSV file. The Query Settings pane will display on the right but you can toggle it by selecting Query Settings on the View tab of the Power Query Editor.


Data Transformation and Processing

Now that we have added our data source, we change and shape the data to meet our needs. When you shape data in Power BI, you are providing Power Query Editor step-by-step isntructions for adjusting the data while loading and presenting it. The original data source remains unaffected while this view of the data is affected.

Shaping and querying the data can mean transforming the data, such as renaming columns or tables, removing rows or column, changing data types, or reodering columns. Each change that you do is captured sequentially by Power BI under the Applied Steps in the Query Settings pane. Everytime this query connects to this data source, the steps are carried in the same order and in the same way, so the data is always in the same shape that you wanted it for. This shaping process occurs when you use the query in Power BI Desktop or when someone uses the shared query.

You will notice that the Applied Steps in Query Settings already has some steps. You can click and highlight each step to see its effect in the Power Query Editor on the data. The first step was to get the data source, which was uploaded from a CSV file. Once it uploaded the data, it promoted the headers that were in the CSV file to headers in the table itself. And lastly, because we allowed the Data Type Detection to be based on the first 200 rows, Power Query Editor looked at the first 200 rows for each column and changed the data type for each column through automatic recognition. For example, it turned the "unMember" column into True/False data type and the Populations to Whole Numbers.
