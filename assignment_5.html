Introduction

Power BI is an interactive data visualization software that was developed by Microsoft with the focus on business intelligence.

Power BI is used for analyzing and visualizing raw data to present information. It combines business analytics, and data visualization to help businesses understand their data and make decisions from it. The core part of Power BI is the interactive dashboard that can be created from the data. The raw data is uploaded and then analyzed or transformed as needed and loaded. From there, the users can create many different kinds of visualizations and dashboards. Power BI has multiple components such as Power BI Desktop, Service and Mobile but we will focus on Desktop.

It is a powerful tool that is very flexible to meet business needs. The interactivity between all of the visuals and data and how the data can be manipulated makes Power BI stand out from the other visualization tools. With features such as Power Querying, DAX (Data Analysis Expression), Visuals and Service, Power BI makes for an excellent choice to present data in different ways all the way from very simple to complex.

Follow along with me and I will show you how to create a basic visual in Power BI!

Let's Get Started

System requirements

  • Windows 10, Windows Server 2012 R2, Windows Server 2012, Windows 8, Windows 8.1, Windows Server 2016, Windows Server 2019, Windows 11
  • Microsoft Power BI Desktop requires Internet Explorer 10 or greater.
  • Microsoft Power BI Desktop is available for 32-bit (x86) and 64-bit (x64) platforms.

To get started, be sure that your system meets the requirements outlined above. First, let's download Power BI. In this tutorial, we will be using Power BI Desktop. Go to Power BI Desktop Download, and select Download Free to download from the Microsoft Store or for other download options, select See download or language options which will give you setup.exe files.

While Power BI is downloading, make sure to create a Microsoft Account. If you already have one, please sign in. This will allow you to publish your report later.

Once you finish downloading and going through the install process, you should have Power BI Desktop correctly installed and ready to go! Don't forget to sign in! This will be important later.

First Time Launch

When you first open Power BI Desktop, it displays the Home Page. From here, you can start with a Blank Report or start with a Data Source. There will also be a section for Recommendations that you can select. The section under will display all recent Reports that you've been working on.

For this example, we will go ahead and select Blank Report. This will take us to a blank report and default report screen of Power BI. On the left side, there are icons for Power BI Desktop Views: Report, Data, Model, and DAX Query View. You can change the view at any time by clicking on the icons and is indicated by the vertical line to the left of the icon.

The Report view is the default view for Power BI reports.

For this example, we won't be worrying about Model View or DAX Querying View. You can find more detailed information about Model View and DAX Querying View on the Learn Microsoft Webpage.



Connecting to the Data

Now that we have Power BI Desktop installed and we've looked at the Home Page and Blank Report views, we'll go ahead and connect some data!

To see all the possible types of data sources that we can upload to Power BI Desktop, select Get Data > More... in the Home tab, and in the Get Data window. Scroll through the list to explore all possible options!

For our example, let's use a 2024 Population Projections by Country dataset from Kaggle! Click HERE to download the .csv file.

Let's imagine we are a data analyst that is trying to look at world populations. You have a dataset that contains historical data as well as projected populations all across the world. You want to use this data to help you and your peers understand what these trends look like. Using the provided .csv file above, we can start populating our Power BI report with actual data.

On the Power BI Desktop Home tab, select Get Data > Text/CSV. From there, open the 2024Populations.csv file. Allow a moment for the Power BI to upload the file.


Another window will appear showing a preview of the data and a title 2024Populations.csv. This window is just previewing the data and giving you an idea as to what your data will look like in a Power BI table. At the top, you will see dropdown options for File Origin, Delimiter, and Data Type Detection. We will leave those as default options and let Power BI auto detect what type of data it will process and the data type that each column has.

From here, we can either Load the data as is, or Transform Data to make changes in the table before we load it in the report.

We will go ahead and click Transform Data to look over the data that Power BI has uploaded and confirm that we are happy with the data types and the stucture that our data is in.


When you select Transform Data, the Power Query Editor Launches, with a view of the table from our CSV file. The Query Settings pane will display on the right but you can toggle it by selecting Query Settings on the View tab of the Power Query Editor.


Data Transformation and Processing

Now that we have added our data source, we change and shape the data to meet our needs. When you shape data in Power BI, you are providing Power Query Editor step-by-step instructions for adjusting the data while loading and presenting it. The original data source remains unaffected while this view of the data is affected.

Shaping and querying the data can mean transforming the data, such as renaming columns or tables, removing rows or column, changing data types, or reordering columns. Each change that you do is captured sequentially by Power BI under the Applied Steps in the Query Settings pane. Everytime this query connects to this data source, the steps are carried in the same order and in the same way, so the data is always in the same shape that you wanted it for. This shaping process occurs when you use the query in Power BI Desktop or when someone uses the shared query.

Querying Steps

You will notice that the Applied Steps in Query Settings already has some steps. You can click and highlight each step to see its effect in the Power Query Editor on the data. The first step was to get the data source, which was uploaded from a CSV file. Once it uploaded the data, it promoted the headers that were in the CSV file to headers in the table itself. And lastly, because we allowed the Data Type Detection to be based on the first 200 rows, Power Query Editor looked at the first 200 rows for each column and changed the data type for each column through automatic recognition. For example, it turned the "unMember" column into True/False data type and the Populations to Whole Numbers.


Data Types

If we need to change the data type of a column, select the column or column we want to change. You can either right-click a column header, and select Change Type and choose a new data type from the menu, or drop down the list next to Data Type in the Transform group in the Home tab, and select a new data type. You can also change the data types by clicking the data type icon next to the Column Headers.




In our example, we'll leave all of the data types alone because the data we provided is fairly clean and consistent that Power BI Desktop was able to automatically detect the intended data types that work well for the data.

Transforming Data

We will go ahead and transform the data. For example, let's go ahead and sort the data by having the pop2030 column Sort Descending and let's also delete the unMember column too by right-clicking the column header and clicking the appropriate option.


You will see in the Applied Steps in the Query Settings that your last 2 steps have been applied to the query. You sorted a row by sorting the pop2030 by descending and Removed Columns by deleting unMember. You can click back and forth between each step to see what happened to that data at that step.


Once you finish transforming your data and shaping it into a way that you want, you are ready to start building reports! Let's apply these changes in the Power Query Editor and load them into Power Bi Desktop by selecting Close & Apply from the Home tab of the ribbon. If you want to keep the Power Query Editor open, you can just click apply.

Creating Reports

Now that you've finished transforming your data and applying your query, you are ready to build your report and create visuals! This is the Report View and it has the following main parts:


  1. The tool ribbon at the top shows common tasks associated with reports and visualizations.
  2. The canvas area in the middle which is where all visualizations will be created, edited and arranged.
  3. The Pages tab at the bottom, which lets us add pages and navigate between them in the report.
  4. The Filters pane, where the visualizations can get filtered based on the data present.
  5. The Visualizations pane, where you can add, change, or customize visualizations, and apply drillthroughs.
  6. The Format pane, where you design the report and visualizations
  7. The Data pane, which shows the available data in the query from our data. You can drag any data onto the canvas, the Filters pane, or the Visualizations pane to create and modify visualizations.

You can expand and collapse the Filters, Visualizations, and Data panes by selecting on the double arrows at the top of the panes. This gives you more space to work with.

Saving and Publishing your New Report

Saving your Report

Congratulations! You've made your own Power BI Desktop Visuals. Now, how can you share this with peers or colleagues? It is very simple! Let's give it a go.

Start by saving your masterpiece somewhere on your system and name it Country Population 2030 Prediction.


Publishing your Report

To publish your report to Power BI, navigate to the File tab at the top left. From there, select Publish > Publish to Power BI. From the Publish to Power BI Window, select My workspace and click Select.


Give Power BI Desktop some time to process the request. Once finished, it will give you a status on whether it passed or not! Go ahead and click Open [Report Name].


Your finished Report

Great! Your report has been complete and is now published on Power BI! It is still interactive and informative just like it was in Power BI Desktop. From the browser view here, you have multiple options:

  1. The File ribbon allows you to make copies, download, print or embed the report. Embedding it allow you to add them to HTML files or other websites without having to send the user to Power BI itself.
  2. The Export ribbon allows you to analyze the report in Excel, view it in Powerpoint or as a PDF.
  3. The Edit ribbon allows you to make edits to the Report live on Power BI without having to open the Report in Power BI Desktop.
  4. The Filters pane allows you to filter and control the visual based on the available data parameters that are in the report.

Go ahead and explore all the the different buttons and options to see how you can interact with the visual. With everything else, you can learn from just messing around!
